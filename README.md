# SQL Agent Optimizer (Telegram + DeepSeek + n8n)

## Проблема и Решение

### AS IS
Долгое SQL-ревью: 30–60 минут на проверку одного запроса экспертом.

100% зависимость от человека: качество ревью варьируется в зависимости от сеньорности и настроения ревьюера.

Узкое горлышко (bottleneck): один разработчик может проверить не более 5–10 запросов в день.

### TOBE
Telegram-бот – пользователь отправляет SQL, получает оптимизированный запрос + оценку за 10–15 секунд.

Автоматическая оптимизация через DeepSeek – переписывает запрос: индексы, JOIN-стратегия, устранение антипаттернов.

LLM-as-a-Judge – выставляет оценку 1–10 по 10 критериям (индексы, фильтры, оконные функции, план выполнения и др.).

Логирование в Google Sheets – все запросы, оценки, пользователи сохраняются для анализа.

### Ожидаемый эффект

| Метрика | До (AS IS) | После (TO BE) | Экономия |
|---------|------------|---------------|----------|
| Время ревью одного SQL | 45 минут | 15 секунд | 99,4% |
| Пропускная способность (запросов/день) | 5–10 | не ограничена (бот работает 24/7) | кратное увеличение |
| Зависимость от эксперта | 100% | 0% | автономное улучшение SQL запросов |

## Архитектура
Система использует 4+ инструмента:
1. Telegram Trigger: Вход пользователя.
2. DeepSeek API (Optimize): Оптимизация кода.
3. DeepSeek API (Review): Оценка оптимизированного кода.
4. Google Sheets: Хранение истории и логов.
5. Telegram Response: Отправка результата пользователю.

## Как запустить
Для локального использования необходимо:

1. Скачать n8n (Cloud или Local).
2. Импортировать файл [workflow.json](./workflow.json) через меню Import.
3. Настроить Credentials:
- В настройках n8n нужно создать Credentials для:
     - Telegram Bot (Token from @BotFather)
     - Google Sheets (OAuth или API Key)
     - DeepSeek API (API Key)
4. Запустить Workflow.
